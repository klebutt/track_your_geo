### 2. Gap analysis (vs GEO Tracker Technical Core spec)
Full analysis saved in [[gap-analysis]]. Key findings:

- Multi-LLM fan-out, citation graph, sentiment scoring, scheduled runs not yet built
- Recommendations infrastructure exists but is disabled
- SQLite/Vite divergences from spec are acceptable

### 3. Multi-LLM citation research
Researched which LLM endpoints return source citations:

| Provider | Model | Approach |
|---|---|---|
| Perplexity | `sonar-pro` | `citations[]` always returned, OpenAI-compatible |
| Gemini | `gemini-2.5-flash` | `google_search` tool → inline annotations |
| OpenAI | `gpt-4o`+ via Responses API | `web_search` tool → `url_citation` annotations ($10/1K — skip for now) |
| Claude | `claude-sonnet-4-6`+ | `web_search_20250305` tool → citations (skip for now) |

**Decision:** Add `sonar-pro` + `gemini-2.5-flash` alongside existing `gpt-4o-mini`.

## Decisions made

| Decision | Rationale |
|---|---|
| Deploy Railway + Vercel | Free tier, minimal config, FastAPI/Vite are a natural fit |
| Add Perplexity sonar-pro first | Cheapest, easiest, citations always returned |
| Add Gemini 2.5 Flash second | Good signal, low token cost (watch grounding cost per query) |
| Skip OpenAI web search for now | $10/1K tool calls — too expensive for prototype |
| Use OpenSpec for Cursor handoff | Already in repo; structured proposal/design/tasks is what Cursor needs |
| Don't add Obsidian to Cursor context | Adds fragility; everything Cursor needs should live in the repo |
