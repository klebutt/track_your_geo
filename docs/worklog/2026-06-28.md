## Async runs + production fixes (afternoon)

### Problem

Full pilot runs (10 queries × 3 models = **30 probes**) take **~4–8 minutes**. The original synchronous `POST /api/runs` held the HTTP connection open with no response bytes, causing **“Failed to fetch”** in the browser after ~2 minutes.

A later failure mode: a single **Gemini** error (e.g. free-tier **429 rate limit**) aborted the entire run and surfaced as *“Analysis run failed on the server.”*

### Solution

| Layer | Behavior |
|--------|----------|
| API | `POST /api/runs` creates run with `status: running`, returns immediately, probes in `BackgroundTasks` |
| API | Each successful probe commits to SQLite incrementally (poll shows growing `query_results`) |
| API | Per-model probe errors logged in `usage_log` (`phase: probe_error`); run still **completes** with partial results |
| API | LiteLLM `completion_cost` dict breakdown normalized via `total_cost` |
| UI | Polls `GET /api/runs/{id}` every 3s; button shows `Running probes… (N complete)` |
| UI | Warns when probe errors present (e.g. Gemini quota) |

### Validated on production (SDL Surveying)

- Run completed with **24/30** probes (OpenAI + Perplexity); **6 Gemini failures** as expected on free tier.
- User plan: move Gemini to paid tier when ready, or temporarily omit from `TYGEO_ENABLED_PROBES`.

### Operational notes

- **Gemini free tier:** ~20 requests/min for `gemini-2.5-flash` — easy to hit on 30-probe runs. Symptoms: `probe_error` entries in `usage_log`, missing Gemini rows in results table.
- **Workaround:** `TYGEO_ENABLED_PROBES=gpt-4o-mini-search-preview,perplexity/sonar-pro` until billing enabled.
- **Run cost:** ~$0.50–0.60 per full 3-model pilot (varies by brand/queries).
